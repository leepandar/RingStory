## 1. 项目概述

### 1.1 产品背景
在数字时代，家庭成员的照片分散在各个成员的手机中，缺乏统一的汇集、整理与情感沉淀工具。现有产品要么只做存储，要么只做社交，无法满足家庭内部“共同记录、按时间脉络浏览、并智能生成回忆”的深层需求。

### 1.2 产品定位
RingStory是一款基于微信生态的私密家庭共享相册工具。它不仅仅是一个云盘，更是家庭的“数字年轮博物馆”——时光为轮，记忆成书。通过**时间轴**、**年轮图谱**和**年轮放映室**，将零散的照片编织成家庭独有的成长史诗。

### 1.3 核心价值
1.  **聚合**：打破设备壁垒，全家人的照片自动汇集一处。
2.  **叙事**：通过照片备注和故事线，让照片“开口说话”。
3.  **发现**：通过AI算法，自动生成月/季/年"记忆切片"，重温感动瞬间。

### 1.4 竞品分析

| 竞品 | 核心定位 | 优势 | 劣势 | 差异化机会 |
| :--- | :------- | :--- | :--- | :----------------- |
| **腾讯相册管家** | 个人云相册备份 | 微信生态无缝集成、用户基数大、免费存储空间充足 | 家庭共享能力弱，缺少多人协作的相册管理；无情感化回顾功能 | 以家庭为单位的沉浸式记忆产品，而非个人存储工具 |
| **亲宝宝** | 亲子成长记录 | 垂直场景深耕（宝宝成长），用户粘性高，有成熟的内容运营体系 | 聚焦亲子场景，不适合多代同堂的"全家庭"覆盖；记录形式偏日记 | 覆盖夫妻、长辈、子女的全家庭场景；创新"年轮图谱"时间聚类与年轮放映室 |
| **Google 相册** | AI 智能相册 | 人脸识别成熟、搜索体验优秀、无限制高质量存储历史 | 国内不可用；家庭共享需使用 Google 账号体系，长辈门槛高 | 基于微信生态的零门槛邀请；符合中国家庭使用习惯的分享与互动体验 |
| **时光相册 / 一刻相册** | 云端相册与共享 | 百度/字节系云存储支撑；一刻有无限存储策略 | 偏向个人存储；家庭化设计薄弱，社交互动功能单一 | 深度家庭关系链设计（管理员/成员/只读），融合家族叙事与智能化 |

### 1.5 商业化策略（待验证）

| 层级 | 定价猜想 | 核心权益 | 适用场景 |
| :--- | :------- | :------- | :------- |
| **免费版** | 0元 | 存储上限1000张照片（压缩后），视频回顾仅支持预览 | 轻体验、低频率使用家庭 |
| **家庭版** | 19元/月 或 199元/年 | 存储上限10万张照片 + 原图存储 + 完整视频回顾生成 + 5个家庭成员 | 核心目标家庭 |
| **尊享版** | 39元/月 或 399元/年 | 不限存储 + 家庭成员不限 + 专属视频模板 + 客服优先响应 | 多成员、高频使用的大家庭 |

- **付费点**：存储空间扩容、原图存储、年轮放映室视频生成（视频合成有第三方 API 成本）、专属视频模板与音乐。
- **冷启动策略**：MVP阶段全免费，仅设置存储软上限（如10万张）。达到 MAU 5万后开启付费模式。
- **成本兜底**：初期单用户存储成本由 OSS + CDN 构成，免费额度内由运营预算覆盖，超出部分通过付费转化对冲。

### 1.6 成功指标（北极星与关键行为指标）

| 指标类型 | 指标名称 | 定义 | MVP 目标值 |
| :------- | :------- | :--- | :--------- |
| **北极星指标** | 家庭周活跃度（WAU_Family） | 每周至少有一次上传/浏览/评论/点赞行为的家庭占比 | >= 40% |
| **留存指标** | 7日家庭留存率 | 创建家庭后第7天仍有成员访问的家庭比例 | >= 60% |
| **核心行为** | 周人均上传照片数 | 活跃成员每周上传的照片数量中位数 | >= 5张 |
| **情感指标** | 备注渗透率 | 有备注的照片数 / 总照片数 | >= 15% |
| **传播指标** | 邀请转化率 | 收到邀请链接后完成加入的比例 | >= 50% |
| **功能指标** | 回顾分享率 | 年轮放映室生成后被分享到微信的比例 | >= 30% |

## 2. 用户角色与画像

| 角色                     | 典型画像                                            | 核心诉求                                                     |
| :----------------------- | :-------------------------------------------------- | :----------------------------------------------------------- |
| **家庭管理员（创建者）** | 家庭中的年轻人（如80/90后父母），喜欢拍照记录生活。 | 主导创建家庭，管理成员权限，维护相册整洁，撰写家庭故事。     |
| **普通家庭成员**         | 配偶、长辈、孩子。操作能力参差不齐。                | 便捷上传（尤其要简单）、浏览家人动态、查看回忆，无需复杂操作。 |
| **只读成员（特殊角色）** | 年迈的长辈，担心误删数据。                          | 仅允许查看和评论，不允许上传或删除，保护数据安全。           |

### 2.1 空状态场景与非 happy-path 体验
- **未加入任何家庭的用户**：首页展示引导页，包含两张插画卡片——"开启家庭年轮"（带大号CTA按钮，文案："刻录第一张照片，开启家的第一圈年轮"）和"加入已有的年轮"（输入邀请码）。不展示空白瀑布流。
- **刚创建但无照片的家庭**：展示空状态插画 + 引导文案"还没有照片，快上传第一张吧" + 突出的拍照/上传按钮，附带示例："试试上传一张全家福 ✨"。
- **邀请链接过期**：被邀请人打开链接后看到"邀请链接已过期"提示页，提供"联系管理员重新邀请"按钮（点击后复制家庭名称，引导通过其他渠道联系管理员）。
- **邀请链接已被使用**：展示"该邀请已被其他成员使用"提示，提供"申请加入"按钮（向管理员发送入群申请）。
- **网络异常页面**：全局网络异常兜底组件，展示离线状态 + "重试"按钮，缓存已浏览过的照片缩略图供离线查看。

### 2.2 适老化与无障碍设计
- **大字模式**：设置中提供"关怀模式"开关，开启后全局字号放大 140%，按钮最小触摸区域 >= 48px。
- **语音输入备注**：备注编辑框支持语音转文字输入（调用微信小程序同声传译插件），降低长辈打字门槛。
- **一键上传**：为"只读成员"以外的长辈角色提供"拍照即上传"快捷入口（首页固定悬浮按钮），跳过后台压缩选择步骤，使用默认智能压缩。
- **图片描述**：每张照片详情页支持添加音频备注（最长 60 秒录音），方便视力不佳的长辈用语音讲述照片故事。
- **色彩无障碍**：所有交互状态（选中/悬停/出错）不使用颜色作为唯一传达方式，同时配合图标或文字变化。

## 4. 详细功能需求说明

### 4.1 用户登录与授权
- **4.1.1 登录流程**
    1.  用户进入「RingStory」小程序，调用 `wx.login` 获取 `code`。
    2.  后端服务通过 `code` 换取 `openId` 和 `session_key`。
    3.  系统判断是否为首次登录：若首次，引导用户授权获取手机号（绑定唯一身份）；若非首次，直接进入首页。
- **4.1.2 用户信息**
    - 头像：默认使用微信头像，允许用户更换。
    - 昵称：默认使用微信昵称，允许修改。
    - 身份标识：系统自动标记为“家长”、“长辈”、“孩子”（仅用于展示，不强制）。

### 4.2 家庭核心模块
- **4.2.1 创建家庭**
    - 入口：首次登录后“创建我的家庭”按钮。
    - 字段：家庭名称（必填，2-20字符）、家庭封面图（默认系统随机风景图，可上传）、家庭所在地（选填，用于时区校准）。
    - 逻辑：创建者自动成为**超级管理员**，拥有最高权限。
- **4.2.2 加入家庭**
    - 入口：点击他人分享的邀请链接/扫码。
    - 流程：进入加入页 -> 展示家庭名称、家庭成员数 -> 点击“加入” -> 若家庭设置为“需审核”，则等待管理员通过；若为“直接加入”，则立即生效。
- **4.2.3 家庭成员列表**
    - 展示所有成员头像、昵称、角色（管理员/成员/只读）。
    - 管理员权限：可移除成员（移除时提示是否同时删除该成员上传的照片，默认选择“否”）、可转让家庭所有权。

### 4.3 邀请机制详细设计
- **4.3.1 生成邀请**
    - 入口：家庭设置页 -> 邀请家庭成员。
    - 生成规则：`https://xxx.com/invite?familyId={id}&token={加密串}`。加密串包含家庭ID、生成时间戳和随机盐值，有效期默认为 **7天**。
    - 分享形式：支持生成带参数的小程序码（分享给微信好友）和复制链接（分享到其他渠道）。
- **4.3.2 邀请状态管理**
    - 未使用：展示“待加入”。
    - 已使用：展示“已加入”。
    - 过期：展示“链接已过期”，支持管理员“重置链接”。

### 4.4 通知系统设计（品牌名：年轮回声）
- **4.4.1 通知类型定义**

| 通知类型 | 触发条件 | 推送对象 | 推送渠道 |
| :------- | :------- | :------- | :------- |
| 新成员加入 | 用户通过邀请链接成功加入家庭 | 所有现有成员 | 微信订阅消息 + 小程序内红点 |
| 照片被点赞 | 某成员点赞了用户上传的照片 | 照片上传者 | 微信订阅消息 + 小程序内红点 |
| 新评论/回复 | 某成员评论照片或回复了用户的评论 | 照片上传者 / 被回复者 | 微信订阅消息 + 小程序内红点 |
| 回顾生成完成 | 年轮放映室制作完毕 | 家庭全体成员 | 微信订阅消息 + 小程序弹窗 |
| 照片违规拦截 | 内容安全检测命中违规 | 家庭管理员 | 微信订阅消息 |
| 存储空间不足 | 家庭存储用量超过 80%/90%/100% | 家庭管理员 | 小程序内通知 + 短信（可选） |

- **4.4.2 通知偏好设置**
    - 入口：个人设置页 -> 通知设置。
    - 每个通知类型支持独立开关（开启/关闭），默认全部开启。
    - 提供"免打扰时段"设置（如 22:00~08:00 不推送），支持自定义时段。
    - 提供"一键静音所有通知"开关。
- **4.4.3 小程序内通知中心**
    - 首页右上角铃铛图标，带未读角标。
    - 通知列表按时间倒序排列，已读/未读状态区分（已读灰色、未读高亮）。
    - 点击通知跳转至对应上下文：点赞通知 -> 照片详情页；评论通知 -> 评论置顶并高亮；回顾通知 -> 回顾播放页。
    - 支持"全部标记为已读"和"左滑删除单条通知"。
    - 历史通知保留 30 天，超出后自动清理。


### 4.5 相册内容管理（核心）
- **4.5.1 刻录上传（照片上传）**
    - **入口**：首页底部大按钮“+”或相册页“上传”按钮。
    - **选择**：支持从手机相册多选（品牌引导文案："选择照片，就是刻录一段时光。"）（最多单次50张，防止内存溢出）。
    - **压缩策略**：
        - 提供开关：“原图上传”（耗流量占空间）与“智能压缩”（默认开启，压缩至长边为1080px，质量80%，保留EXIF信息）。
    - **上传逻辑**：
        - 采用分片上传（每片1MB），支持断点续传。
        - 显示上传进度列表（文件名+进度条）。
        - **关键点**：读取照片的 `EXIF DateTimeOriginal`（拍摄时间）。若无法读取，则使用文件的修改时间；若修改时间异常，则默认使用当前服务器时间（但提醒用户手动修改）。
- **4.5.2 照片展示（时间轴模式）**
    - 采用**无限瀑布流**，按拍摄时间 **倒序排列**（最新的在最上面）。
    - 日期分割线：例如“今天”、“昨天”、“2026年7月15日”。
    - 展示元素：缩略图（3列网格）、上传者头像（缩略图右下角小图标）。
- **4.5.3 照片详情页**
    - 点击缩略图进入全屏预览，支持左右滑动切换上下张。
    - 显示信息：
        - 照片大图（支持双指缩放）。
        - 拍摄时间（具体到秒）。
        - 上传者昵称与上传时间。
        - 地理位置（若有）。
        - 备注内容（若有）。
    - 操作按钮：点赞、收藏（标记为“特别回忆”）、下载到本地、删除（仅限本人或管理员）。

- **4.5.4 EXIF 隐私保护**
    - **位置信息开关**：照片详情页地理位置默认对"非上传者"隐藏，查看者需点击"显示位置"按钮手动确认。
    - **隐私配置**：上传时提供"是否保留位置信息"选项（默认"移除位置"，保留后其他成员可见）。
    - **批量操作**：管理员可在家庭设置中开启"全局隐藏位置信息"，所有家庭成员的地理位置在上传时自动剥离。


### 4.6 “年轮图谱”功能
- **4.6.1 概念定义**
    “年轮图谱”不是简单的文件夹，而是一种**递归的、非线性的时间聚类可视化**。根节点是家庭本身，第一层是**年份**，第二层是**季节（春3-5、夏6-8、秋9-11、冬12-2）**，第三层是**月份**，叶子节点是具体的**日/事件**。
- 进入年轮图谱页时，顶部展示品牌引导文案："循着年轮，回溯家的四季。"
- **4.6.2 交互与展示**
    - 入口：首页顶部Tab“年轮图谱”。
    - 界面设计：类似思维导图的横向展开布局。
    - 点击“2026年”-> 展开为四个季节分支 -> 点击“夏季” -> 展开为 6、7、8月 -> 点击“7月” -> 展示该月所有日期的卡片预览。
    - 节点标识：若该节点下照片数量 > 0，节点显示为绿色/高亮并带有数字角标（如“12张”）。
- **4.6.3 生成算法**
    - 后端定时任务（或实时）解析照片的拍摄时间。
    - 按照 `Year -> Season -> Month -> Day` 构建多叉树数据结构。
    - 若某个月份的照片少于3张，为了视觉美观，自动将相邻月份合并为“季度碎片”节点。

### 4.7 “撰写故事”功能（品牌名称：刻录此刻）
- **4.7.1 功能描述**
    允许用户为单张照片添加丰富的备注，用来完善照片背后的故事。
- **4.7.2 编辑能力**
    - 支持富文本编辑（加粗、斜体、换行）。
    - 支持添加“谁在照片里？”（@家庭成员，如 `@爸爸`、`@宝宝`，可被点击跳转至成员主页）。
    - 支持添加“在哪里？”（调用地图选点，存储经纬度和POI名称）。
- **4.7.3 展示样式**
    - 在照片详情页下方，以卡片形式展示备注内容。
    - 在“年轮图谱”的叶子节点（某一天）下，如果当天所有照片都有备注，展示一个“📖 今日故事完结”徽章。
- **4.7.4 备注协作模型**
    - **编辑权限**：上传者和管理员可编辑/删除备注；普通成员可查看但不可修改他人备注。
    - **补充备注**：若普通成员想补充信息，可在备注下方以"子评论"形式添加补充内容，展示为"某某的补充"。
    - **版本记录**：每次编辑备注后自动生成历史版本（最多保留最近 5 个版本），支持回滚。
    - **冲突处理**：多人同时编辑同一备注时，后保存者会看到"备注已被他人修改，请刷新后重试"提示。


### 4.8 “年轮放映室”功能
- **4.8.1 回顾类型与生成时间**
    - **每月回顾**：次月1日 08:00 自动生成。选取上月**点赞数最多**的9张照片，生成拼图海报。
    - **每季回顾**：次季首月1日（1/4/7/10月）生成。选取本季照片中的**人脸出现频率最高**的人物作为“本季之星”，配合时间流逝动画生成15秒短视频。
    - **年度回顾**：次年1月1日 00:10 生成。选取本年**所有月份**的代表作，生成一个约1分钟的“家庭年度大片”视频（带背景音乐和转场特效）。
- **4.8.2 回顾查看与分享**
    - 在首页设有“时光机”Tab，按年份排列已生成的回顾。
    - 用户可一键将回顾海报/视频**保存到手机相册**，或分享给微信好友/朋友圈。
- **4.8.3 算法逻辑细化**
    - 排除模糊照片（利用OpenCV或腾讯云图片检测接口评估清晰度）。
    - 去重：若连续多张照片相似度>90%（感知哈希算法），只保留1张参与回顾。
- **4.8.4 视频合成技术选型评估**

| 评估维度 | 腾讯云 VOD 视频处理 | 阿里云 MediaAI | 自研 + FFmpeg |
| :------- | :----------------- | :------------- | :------------ |
| 成本 | 按分钟计费，约 0.032元/分钟 | 按分钟计费，约 0.028元/分钟 | 仅需服务器 FFmpeg + GPU 成本 |
| 素材上传耗时 | 需上传原图到 COS，耗时较长 | 需上传原图到 OSS，耗时较长 | 本地处理，无上传耗时 |
| 定制化程度 | 提供模板引擎，可定制字幕/音乐/转场 | 模板丰富，支持智能匹配 | 完全可控，但开发成本高 |
| 依赖风险 | 高（第三方 API 可能变更定价策略） | 高 | 低（完全自主可控） |
| 画质 | 支持 1080p 输出 | 支持 1080p 输出 | 可达 4K |

    - **MVP 推荐方案**：初期接入腾讯云 VOD（利用微信生态集成优势），同步自研基于 FFmpeg 的轻量视频合成服务作为备选。
    - **降级方案**：若第三方服务不可用，回退到生成拼图海报（静态图片）替代视频回顾。

### 4.9 全局搜索功能
- **4.9.1 搜索范围**
    - 按**日期**搜索：支持自然语言输入（"去年春节""2025年元旦""上个月"）及日期选择器。
    - 按**人物**搜索：基于人脸聚类结果，点击人物头像直接展示该成员出现的所有照片。
    - 按**地点**搜索：基于照片地理位置/POS名称，支持关键词模糊匹配。
    - 按**备注**搜索：全文检索照片备注内容（含富文本纯文本提取）。
- **4.9.2 搜索交互**
    - 首页顶部搜索框（常驻），支持搜索历史记录展示（最近 10 条）。
    - 搜索结果按拍摄时间倒序排列，支持筛选（仅显示有备注的照片/仅显示收藏的照片）。
    - 支持多条件组合搜索：如"2025年" + "@宝宝" + "北京"。

### 4.10 影集与标签
- **4.10.1 影集（手动专辑）**
    - 入口：相册页 Tab "影集"。
    - 创建：点击"新建影集" -> 输入名称（必填，2-20字符） + 封面图（从已有照片中选择） -> 从相册中选取照片加入。
    - 管理：影集所有者/管理员可增删照片、修改影集名称和封面、删除影集（删除影集不影响原照片）。
    - 协作：所有成员可浏览影集中的照片，管理员可设置为"允许成员添加照片到该影集"。
- **4.10.2 照片标签**
    - 支持为单张/多张照片批量添加标签（如"旅行""生日""美食""毕业"）。
    - 标签支持自定义创建，系统自动推荐高频标签。
    - 标签页聚合展示：点击标签展示该标签下的所有照片瀑布流。

### 4.11 人物相册（人脸聚类）
- **4.11.1 人脸检测与聚类**
    - 照片上传后异步触发人脸检测（调用腾讯云/阿里云人脸识别 API）。
    - 检测到的人脸按相似度聚类（相似度 >= 85% 视为同一人），自动生成"人物"实体。
    - 聚类结果需管理员确认并命名（如"爸爸""妈妈""宝宝"），未命名的人物显示为"未标注人物"。
- **4.11.2 人物封面与时间线**
    - 每个命名人物拥有独立封面页：展示人物头像（从照片中裁剪的最佳人脸） + 照片数量 + 时间分布图。
    - 点击人物进入"该人物的照片时间线"，按拍摄时间倒序展示包含该人物的所有照片。
    - 支持人物照片分享：一键生成"宝宝成长集"并分享到家庭群。
- **4.11.3 隐私与手动修正**
    - 成员可手动将自己从某张照片中排除（"这张照片里不标记我"），排除后该照片不再出现在该人物的时间线中。
    - 管理员可合并/拆分聚类结果（如将"小明"和"小明2号"合并为同一人）。

### 4.12 照片批量导入
- **4.12.1 导入来源**
    - **微信聊天记录**：支持从家庭微信群聊记录中批量勾选照片导入（调用微信小程序选择本地图片接口）。
    - **手机相册**：支持按相册夹/按日期范围批量选择（突破 50 张单次限制，上限 500 张/次）。
    - **其他云服务**：Phase 2 考虑支持从百度网盘/阿里云盘/Google 相册导入（需 OAuth 授权）。
- **4.12.2 导入流程**
    - 选择来源 -> 勾选照片（显示已选数量/总数量） -> 确认导入 -> 后台异步处理。
    - 导入过程中提供进度条（已处理 x/n 张），支持"后台运行"继续浏览其他页面。
    - 导入完成后推送通知："已成功导入 xx 张照片到家庭相册"。
    - 去重逻辑：已存在于家庭相册中的照片（基于文件 MD5 比对）自动跳过，在结果中展示"已跳过 xx 张重复照片"。

## 6. 非功能性需求

### 6.1 性能要求
- **首屏加载**：小程序首页（相册瀑布流）在白屏状态下加载时间不超过 **1.5秒**。
- **图片加载**：采用懒加载和WebP格式（若设备支持），缩略图大小控制在 30KB 以内。
- **并发支持**：需支持单家庭最大 **500** 名成员，单家庭照片总数最大支持 **10万** 张。

### 6.2 安全与隐私
- **数据隔离**：严格依据 `family_id` 进行数据分表/分区隔离，严禁跨家庭数据泄露。
- **传输加密**：全站启用 HTTPS，敏感用户信息（手机号）数据库加密存储。
- **内容安全**：上传图片需接入腾讯云/阿里云的内容安全检测（鉴黄/暴恐/政治敏感），违规图片自动拦截并通知管理员删除。

### 6.3 可用性设计
- **操作简化**：针对“上传”流程，必须保证在 **3步以内** 完成（选择照片 -> 确认 -> 开始上传）。
- **异常提示**：上传失败（如网络中断）时，需提供明确的“重试”按钮，且未上传成功的照片要在列表顶部置顶显示。

## 8. 验收标准（关键场景测试）

- **场景1**：用户A创建家庭“幸福一家”，生成链接发给B。B点击链接加入，A在成员列表中看到B，且B拥有上传权限。
- **场景2**：A上传10张拍摄于2025年元旦的照片。在“年轮图谱”视图中，点击2025 -> 冬季 -> 1月 -> 1月1日，能看到这10张照片的缩略图聚合。
- **场景3**：A为其中一张照片添加备注“@宝宝 第一次在雪地里走路，摔了一跤也没哭。”，备注栏显示正确，且@标记可点击。
- **场景4**：在次月1号，A和B均收到系统推送的“月份回顾”，打开后看到的是上个月互动（点赞）最高的9张照片拼接的海报。

- **场景5（负面 - 网络中断）**：A 在上传 30 张照片的过程中切换至飞行模式，上传进度条中断。恢复网络后，已上传的 15 张出现在相册中，中断的 15 张在列表顶部置顶显示"上传失败"状态 + "重试"按钮。
- **场景6（负面 - 权限验证）**：只读成员 C 尝试删除一张照片，删除按钮置灰不可点击；若尝试通过 API 直接删除，返回 403 错误且触发审计日志。
- **场景7（负面 - 邀请失效）**：A 生成邀请链接后第 8 天（有效期 7 天），B 点击该链接，页面展示"邀请链接已过期"，并提供"通知管理员重新邀请"按钮。A 收到通知后可在邀请列表重置链接。
- **场景8（负面 - 存储超限）**：家庭相册存储已达 10 万张上限，A 尝试上传新照片。前端弹出提示"家庭存储空间已满，请升级套餐或删除旧照片"，上传按钮置灰。升级后自动恢复上传能力。
- **场景9（并发冲突）**：A 和 B 同时编辑同一张照片的备注。A 先保存（成功），B 后保存时弹出提示"备注已被他人修改，请刷新后查看"，并展示 A 修改后的版本。

## 9. 客服与用户反馈

### 9.1 反馈入口
- **小程序内置反馈**：个人设置页 -> "意见反馈"入口，支持文字描述 + 截图上传 + 自动附带设备信息（小程序版本、系统版本、网络类型）。
- **微信客服**：接入微信客服消息能力，用户可在小程序内直接与客服对话，支持发送文字/图片/语音。
- **紧急联系**：在"账号安全"页面显著位置展示客服微信号和服务时间（工作日 09:00~21:00）。

### 9.2 反馈处理流程
1. 用户提交反馈后，系统自动分配工单编号并通过微信订阅消息告知用户。
2. 简单咨询（如"怎么退出家庭"）由 AI 自动回复（匹配 FAQ 知识库，50 个常见问题覆盖）。
3. 复杂问题（如数据恢复、账号申诉）转人工客服，承诺 24 小时内首次回复。
4. 用户可在"我的反馈"页面查看工单状态（待处理/处理中/已解决）和历史记录。

### 9.3 社区与自助帮助
- **帮助中心**：小程序内设 H5 帮助中心，包含图文教程（"如何创建家庭""如何使用年轮图谱"）和常见问题（FAQ）。
- **新功能引导**：每次新功能上线后，在首页弹出轻量引导遮罩（不超过 3 步），用户可点击"跳过"或"不再提示"。
- **运营群**：建立种子用户微信群（上限 500 人/群），产品初期用于收集反馈和 Bug 报告，用户可自愿扫码加入。
